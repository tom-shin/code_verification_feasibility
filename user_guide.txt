1. 윈도우에서  main.exe 실행 한다.

2-1. Input Your Prompt에 script 작성 한다. 
      예를들어 "1부터 100까지 더하는 파이선 코드 작성 해 봐"

2-2-1.
       만약 파일을 분석하고 싶으시면 왼쪽 상단 File 메뉴 선택 후 Open Fold ...을 선택 하시어 분석하고자 하는 프로젝트 폴더를 선택 후 열기 합니다.
       그러면 왼쪽에 해당 폴더를 기준으로 디렉토리가 불러오게 됩니다.
       참고로 Open File ... 메뉴는 아직 미 구현입니다. 하단의 Hide(Show) 버턴을 누르면 toggle 형태로 폴더 구조 화면이 visible, invisible합니다.
2-2-2.
       불러온 폴더를 펼치기 한 후 원하는 폴더 또는 파일을 클릭합니다. 그러면 선택된 폴더 또는 파일이 빨간색으로 표시가 되고 이는 선택되었다는
       의미입니다.
       만약 선택 해제 하고 싶으면 하단의 Deselect 버턴을 누르시면 됩니다.

3. Input Your Prompt 창에 적절한 prompt를 입력합니다. 현재 default로 보시는 것과 같습니다.
   - prompt 창에 있는 내용을 지우고 싶으면 "Clear Prompt" 버턴
   - 사용자가 선택한 Prompt를 저장하고 싶으면 "Save Prompt" 버턴. 그럼 이후 main.exe 실행하시면 항상 마지막 저장한 Prompt가 입력됩니다.
     * prompt를 저장할 때 마다 저장 후 get 할 때 기존 저장된 목록에서 가져오는 것은 현재 미 구현 (구현 상 어렵지는 않습니다)

4. Analyze 버턴을 누른다.
   * 오른쪽 상단에 LLM Model를 선택 하면 해당 모델로 동작 합니다
   * GPT Key 요금은 AI 포털 개발 시 gpt 사용 라이센스 사용했습니다.
     다만 개인 컴에 OPENAI_API_KEY 를 먼저 환경 변수로 설정 해 두셨다면 해당 키가 더 우선입니다.
     하기 코드 흐름.

     openai_key = os.getenv("OPENAI_API_KEY")  # 개인 노트북에 설정된 key 값
     if openai_key is None:
	openai_key = "".join(self.CONFIG_PARAMS["openai_key"]["key"])  # 제가 갖고 있는 license key 값

5. 기타 
  * 오른쪽 하단 Time Out은 LLM에게 요청하고 해당 시간 동안 응답이 없으면 종료 됨.
  * 하단의 popup control은 제가 숨김 기능으로 가져 가는 겁니다. 체크가 안되어 있으면 뭔가 작업 할 때 팝업이 나올텐데 해당 팝업이 모달로 동작하면서
    사용자의 key event를 무시하게 됩니다. 사용자가 이거 저거 막 테스트하면서 버그가 나올 것 같아...ㅎㅎ
    반면 체크되어있으면 사용자 키 이벤트가 동작하게 되며 이 경우 팝업 뒤에 있는 UI 창을 선택하거나 하면서 제어 가능 합니다. 


6. 향후 고려 사항
   - 무료 LLM 찾아서 코드 구현
   - LLM에게 요청 시 smart chunk 알고리즘 구현 (????) 고민 필요....(규모가 클 것 같습니다)
   * 현재는 잘 아시겠지만 LLM 한번에 입력되는 사이즈 제한이 있기에 (토큰 계산 해보면 A4 용지 영문 1Byte 기준으로 대략 4000 라인 정도)
     
    선택한 모든 파일/ 폴더에 대해서 모든 내용을 버퍼에 입력 후 --> 일정 크기(4000라인)로 잘라서 (서로 간 2~3줄 오버랩해서) 여러개의 청크를 만듭니다. 
    chunk_0, chunk_1...chunk_N  --> 각 chunk별로 LLM에 넣어 요약해 달라고 함. 그럼 요약된 s_chunk_0, s_chunk_1 ... s_chunk_N
    --> 이렇게 요약된 chunk를 하나로 만들어 LLM에 던져 주는 방식을 사용하고 있습니다. 
          (recrusive하게 줄여가면서 어느정도 줄여졌다면 그때 LLM에 던집니다)
          너무 줄이면 정보가 사라질 거고....LLM입장에서는 전체를 한번에 받아야 최대한 분석하기 좋을 것 같은데...
          여튼 이런 방법을 지금은 사용하고 있는데 이 부분에 대한 스마트한 알고리즘 개발(??)/ 발굴이 핵심이 될 것 같습니다.

7. 평가 완료되면 main.exe 위치에 Result 폴더가 있을 거고 해당 폴더안에 평가 결과가 .md 형태로 저장이 되어있을 겁니다.
